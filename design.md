# Linc モバイルアプリ設計書

## 概要
読み書きに困りを感じている子どもの保護者向けスクリーニングアプリ。質問に答えることで考えられる要因と相談先の目安を提示するMVPモック。

## ターゲット
- 主：20–30代女性（母親中心）
- 副：父親・支援者

## UXゴール
- 3–7分で完了
- 不安を煽らない
- 責めない
- 希望がある
- 「次に何をすればいいか」が明確

---

## 画面一覧

### 1. Onboarding（オンボーディング）
**目的**: アプリの説明と開始

**コンテンツ**:
- タイトル: 「お子さんの「読む・書く」の困りを整理します」
- 説明文: 「いくつかの質問に答えることで、考えられる要因と相談先の目安をお伝えします。これは診断ではなく、ひとつのスクリーニングです。」
- 所要時間: 「約5分」
- CTA: 「はじめる」ボタン

### 2. STEP01（苦手選択）
**目的**: 苦手な領域を選択

**コンテンツ**:
- 進捗バー
- 質問: 「なにが苦手ですか？」
- カード選択:
  - 書く
  - 読む
  - 書く・読む
- 「次へ」ボタン

### 3. STEP02（一次スクリーニング）
**目的**: 一次スクリーニング質問

**コンテンツ**:
- 進捗バー（1問ずつ表示）
- 設問文
- チェックボックス
- 補助文: 「迷った場合は、より近いものを選んでください」
- 「次へ」ボタン
- 戻る機能（回答保持）

**設問（書く）**:
1. 黒板を写すのが苦手、または遅い
2. 図形や絵を見て同じように書き写すことが苦手
3. マス目や枠から文字がはみ出ることがよくある
4. ひらがなの50音表をサラサラと書くことができない
5. 自分の名前をひらがなで正しく書くことができない
6. 字を書くことを嫌がる

**設問（読む）**:
1. 音読の際、読む行を指で押さえながら読むことがよくある
2. ひらがなの50音表をスラスラと読むことができない
3. 字を読むことを嫌がる
4. 逐次読みをする
5. 文末や助詞を正確に読むことが苦手

### 4. STEP03（詳細設問）
**目的**: 候補要因の詳細スクリーニング

**コンテンツ**:
- 進捗バー（1問ずつ表示）
- 設問文
- チェックボックス
- 「次へ」ボタン
- 戻る機能（回答保持）

**設問（要因別）**:
- 音韻（6問）
- 眼球（3問）
- 運動（4問）
- 視知覚（5問）

### 5. STEP04（結果画面）
**目的**: 結果表示と次のアクション提示

**コンテンツ（順序厳守）**:
1. 要約カード: 「あなたが「{書く/読む/書く・読む}」で困りを感じる背景には「{要因}」が関係している可能性があります。」
2. 支援職カード: 「{ST/OT/両方}による支援をおすすめします。」
3. Primary CTA: 「📍 近隣の支援機関を探す（地図）」
4. Secondary CTA: 「✉️ この結果で問い合わせる」
5. 免責: 「これは診断ではなく、ひとつの目安です。」

## ユーザーフロー

```
Onboarding
    ↓ [はじめる]
STEP01（苦手選択）
    ↓ [次へ]
STEP02（一次スクリーニング）
    ↓ [次へ] × 設問数
STEP03（詳細設問）
    ↓ [次へ] × 設問数
STEP04（結果）
    ↓ [近隣の支援機関を探す] or [この結果で問い合わせる]
地図/問い合わせ
```

## UI基本方針

- iOS Human Interface Guidelines準拠
- 大きな余白 / カード UI / やさしい日本語
- タップ領域44px以上
- 色だけに依存しない情報設計
- 医療的"診断UI"に寄せない
- 情報量を勝手に増やさない

---

## コピー方針

**必須表現**:
- 可能性があります
- ひとつの目安です
- 診断ではありません

**禁止表現**:
- 断定
- 障害という表現
- 将来不安を煽る

---

## データ仕様

- 端末内のみ保存（AsyncStorage）
- クラウドなし
- 戻るで回答保持
- 未回答警告
- 中断復帰
